# CHANGELOG

## 1.0.2 - 2026-07-15

- 修复 WebSocket 模式只安装事件解析器、却没有订阅 `GROUP_MEMBER` Intents（`1 << 24`），导致普通成员进退群事件根本不会下发的问题。
- 启动时保留原有 Intents 并补充成员事件订阅位；若连接已建立，会明确提示重载 QQ 平台或重启 AstrBot。
- README 补充 WebSocket 与 Webhook 的选择、HTTP 回调配置和事件权限说明。
- 增加 Intents 合并回归测试。

## 1.0.1 - 2026-07-14

- 修复从 AstrBot 管理面板安装、更新或热重载后，已有 QQ 官方平台实例没有接入通知回调的问题。
- 修复退群通知错误携带 `GROUP_MEMBER_REMOVE` 事件 ID，可能被 QQ 群消息接口拒绝的问题。
- 补充热重载绑定和退群主动消息的回归测试。

## 1.0.0 - 2026-07-14

- 云云完成首个版本。
- 支持机器人加入群聊时发送自定义打招呼消息。
- 支持普通成员进群欢迎和退群通知。
- 为 AstrBot 原生 QQ WebSocket/Webhook 适配器补充可卸载的成员事件解析桥。
- 三类通知分别提供开关和模板。
- 支持群黑名单、白名单和群名称映射。
- 支持事件字段、运行期昵称缓存和 OpenID 回退。
- 加入 `event_id` 去重和调试日志。
- 加入管理员 `/群通知信息` 指令，便于查看当前群 OpenID、群名称和过滤结果。
